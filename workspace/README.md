# workspace

«Кухня» — всё сырое и несборочное. Astro сюда не заглядывает; в прод напрямую
отсюда ничего не попадает. Корень репо чистый, всё черновое — здесь.

```
drafts/          Статьи в работе.
  ideas.md       Бэклог идей. Одна строка = одна идея. Без frontmatter,
                 без структуры — просто список. Идея «созревает» → заводим папку.
  <slug>/        Папка на статью (русский slug, как думается). Внутри:
    main.md      Рыхлый русский черновик — пишем как думаем, без схемы и метаданных.
                 Рядом кладём всё по этой статье: ресёрч, картинки, заметки.

cv/              Резюме: backbone → reflections → pdf. Свой пайплайн, см. cv/README.md.
```

## Поток статьи (этапы)

```
1. ideas.md                                   идея, одна строка
2. drafts/<slug>/main.md                       русский черновик, свободная форма
3. apps/web/src/content/blog/note-NN-slug/ru.mdx   оформляем RU в схему, draft: true
4. apps/web/src/content/blog/note-NN-slug/en.mdx   перевод EN, translatedFrom: "ru"
5. снимаем draft: true                          выпуск
```

«Кухня» (шаги 1–2) → «витрина» (`src/content/blog`, шаги 3–5). Граница осознанная:
**выпуск — это перенос черновика в коллекцию и оформление по схеме, а не смена
флага.** На «кухне» нет схемы и ограничений; в «витрине» запись проходит
`content.config.ts`, получает номер и рендерится.

## Конвенции витрины

- **Папка на статью:** `blog/note-NN-slug/ru.mdx` + `en.mdx`. Симметрично черновикам;
  обложка и ассеты статьи лежат в той же папке.
- **Связь RU↔EN — общий `articleNumber`.** Номер статьи и так сквозной ритуал бренда;
  он же служит ключом связи между языковыми версиями. Новых полей для линковки не нужно.
- **Маркер перевода:** в EN-версии перевода — `translatedFrom: "ru"` во frontmatter.
  В UI рендерится плашкой «Translated from Russian» со ссылкой на оригинал.
- **Slug/URL:** роуты `blog/[...slug].astro` строят путь из `post.id`, а у вложенной
  папки `id` = `note-NN-slug/ru`. Чтобы язык не задваивался в URL
  (`/blog/note-NN-slug/en`), при первом выпуске парной статьи в обоих
  `[...slug].astro` slug выводить как `post.id.replace(/\/(ru|en)$/, "")`.
  Плоские старые заметки (`note-01-...`) от этой правки не страдают.

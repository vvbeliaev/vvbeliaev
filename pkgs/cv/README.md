# cv

Резюме как **один источник правды → отражения под вакансии**.

```
backbone.yaml    Источник правды. Подробная НАСТОЯЩАЯ биография: всё, что было,
                 с датами, метриками, контекстом. Пункты помечены тегами roles:.
                 Правим только здесь.

reflections/     Отражения — срезы backbone под конкретную роль/язык, в формате
                 rxresume.me (JSON, машинный). Сейчас: {en,ru}-{ai-fullstack,
                 analyst,devops,tpm}.json. Регенерируются из backbone.

pdf/             Готовые PDF-рендеры отражений.

sources/         Сырые импорты из старой репы (me.json, labs.json) — справочно,
                 уже слиты в backbone.yaml, в пайплайне не участвуют.
```

## Почему YAML для backbone, JSON для отражений

Backbone редактируется руками годами — нужны комментарии, многострочные
формулировки, читаемость. YAML это даёт; JSON — нет. Отражения, наоборот,
машинно-генерируемые экспорты — им место в JSON.

## Поток

`backbone.yaml` → (выбрать пункты по `roles:` + переформулировать под вакансию)
→ `reflections/<lang>-<role>.json` → рендер → `pdf/<...>.pdf`.
